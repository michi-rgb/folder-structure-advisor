# フォルダ構成改善提案アプリ 設計計画

## 0. 目的（課題との対応）
散在・無秩序なフォルダ／ファイルを走査し、**内容の重複を最小化した改善後フォルダ構成を提案**する。
実ファイルは一切動かさず（提案+可視化のみ）、before/after をグラフ・mermaid 図で可視化する。

対応する症状：
- A. 探索の困難 → 重複・旧版を洗い出し、意味のある階層に再分類した提案を出す
- B. 信頼性の判断不能 → メタデータの**信頼度**を評価し、当てにできない値（更新日・作成者）に依存しない分析を行う

## 1. スコープ（確定した方針）
| 項目 | 決定 |
|---|---|
| 対象 | Windows ローカル。SharePoint は**同期済みローカルパス**として同じ経路で走査（将来 Graph API を追加できる抽象化） |
| 提案エンジン | ルールベース + LLM補助（**Azure OpenAI**）。キー未設定ならルールベース単体で動作 |
| 実行範囲 | **提案 + 可視化のみ**。実ファイルは動かさない。移動計画は CSV として出力するだけ |
| 形態 | CLI 実行 → HTML レポート（mermaid 図・表）をブラウザで確認 |

## 2. 全体フロー
```
scan（走査）→ enrich（ハッシュ・メタ信頼度）→ analyze（重複/旧版/分類）
→ propose（改善後ツリー・命名規約）→ visualize（before/after mermaid＋グラフJSON）
→ report（HTMLレポート＋移動計画CSV）
```

## 3. モジュール構成
```
folder_advisor/
  __init__.py
  models.py        # FileEntry / DirEntry / 分析結果 dataclass
  scanner.py       # os.walk 走査。SourceBackend 抽象（local / sharepoint-synced）
  enrich.py        # 内容ハッシュ、メタデータ信頼度スコア
  duplicates.py    # 完全重複（ハッシュ）＋ 近似重複（名前類似）
  versioning.py    # 旧版・作業中ファイルの検出ヒューリスティック
  classifier.py    # ルールベース分類（拡張子/トークン/パス）＋ LLM補助フック
  llm.py           # Azure OpenAI ラッパー（任意・失敗時フォールバック）
  proposer.py      # 改善後ツリー生成、命名規約案、移動計画
  visualize.py     # before/after mermaid、グラフ JSON（ノード集約対応）
  report.py        # 自己完結 HTML レポート生成
  cli.py           # 引数解析・オーケストレーション
config.example.toml
requirements.txt
README.md
```

## 4. 各処理の要点

### 4.1 走査（scanner）
- `SourceBackend` 抽象で `LocalBackend` を実装。SharePoint 同期フォルダも同じ `LocalBackend`。
- 収集項目：相対パス／フォルダ名／ファイル名／拡張子／サイズ／作成日時／更新日時。
- 除外設定：`~$` 一時ファイル、`.tmp`、`Thumbs.db`、`.git` 等。大サイズ／件数上限をガード。

### 4.2 メタデータ信頼度（enrich）
課題 B を踏まえ、値を鵜呑みにせず**信頼度**を付与：
- 更新日：開閉で書き換わるため「参考」扱い（低信頼）。
- 作成者：雛形作成者の残存があり得るため「低信頼」フラグ。
- 代わりに **内容ハッシュ（SHA-256）** を重複判定の一次情報にする。
- 各ファイルに `reliability: {mtime: low, author: low, hash: high}` を持たせ、レポートで明示。

### 4.3 重複検出（duplicates）
- **完全重複**：ハッシュ一致でグループ化 → 「正」候補を1つ選び他を統合対象に。
- **近似重複**：ファイル名の正規化（版表記・日付・「コピー」除去）後の類似度で候補提示。

### 4.4 旧版・作業中検出（versioning）
ファイル名パターンで検出（例）：
- 版：`v1 v2 ver1.2 第2版 rev`、日付 `2024xxxx`、`最新 旧 old final 確定`
- 作業中：`（作業中） WIP draft メモ tmp コピー - コピー`
→ 同一系列をまとめ、最新候補と旧版候補を提示（※判断材料として提示、自動削除はしない）。

### 4.5 分類（classifier）＋ LLM補助（llm）
- ルールベース：拡張子・パス上の既存カテゴリ語・ファイル名トークンで一次分類。
- LLM補助（任意）：ファイル名／フォルダ名の**リストのみ**を Azure OpenAI に渡し、
  - 日本語ファイル名の意味カテゴリ推定
  - 命名規約・フォルダ体系案の提案
  - を得る。**ファイル内容は送らない**（社外秘リスク低減）。
- **Azure OpenAI 接続**：`openai` パッケージの `AzureOpenAI` クライアントを使用。
  設定は環境変数／config から取得：
  - `AZURE_OPENAI_ENDPOINT`（例：`https://<resource>.openai.azure.com/`）
  - `AZURE_OPENAI_API_KEY`
  - `AZURE_OPENAI_DEPLOYMENT`（デプロイ名。モデル名ではなくデプロイ名を指定）
  - `AZURE_OPENAI_API_VERSION`（例：`2024-10-21` 等）
  - JSON 応答は `response_format={"type":"json_object"}` で構造化して受け取る。
- 上記いずれか未設定／API 失敗時はルールベース結果のみで完結（graceful degrade）。
- Azure OpenAI の正確な API バージョン・SDK 用法は Microsoft Learn ドキュメントで確認して実装。

### 4.6 提案生成（proposer）
- 目的別＞資料種別＞（必要なら）年度、を基本とした階層テンプレートを提示。
- 重複は「正」1本に集約する移動計画を作成。
- 命名規約案（例：`YYYYMMDD_案件_資料名_版`）を提示。
- **移動計画 CSV**：`現在パス, 提案パス, 分類, 重複/旧版フラグ, 根拠, アクション(移動/統合/据置/要確認)`。

### 4.7 可視化（visualize）
- before / after を **mermaid**（`graph TD` もしくは `flowchart`）で出力。
- 大規模ツリー対策：深さ上限・子ノード数上限で**集約ノード**（「他 N 件」）に折り畳み。
- 併せて **グラフ JSON**（nodes/edges）を出力（他ツール連携・再描画用）。

### 4.8 レポート（report）
自己完結 HTML（外部 CDN 非依存、mermaid はローカル同梱 or 図はサーバサイドで SVG 化を検討）に以下を掲載：
- サマリ（総数・重複件数・旧版件数・削減見込み）
- before/after 図
- 重複グループ表 / 旧版系列表 / 分類別内訳
- 移動計画（CSV へのリンク）
- メタデータ信頼度の注記

## 5. CLI インターフェース（案）
```
python -m folder_advisor scan   --source "D:\共有" [--mode local|sharepoint] --out out\
python -m folder_advisor report --scan out\scan.json --out out\   [--llm]
# もしくは一括：
python -m folder_advisor run    --source "..." --out out\ [--llm] [--max-files N]
```
出力：`out/scan.json`（走査生データ）／`out/analysis.json`／`out/move_plan.csv`／`out/report.html`

## 6. 依存関係（最小）
- 標準ライブラリ中心（os, hashlib, pathlib, csv, json, dataclasses, html）
- 任意：`openai`（Azure OpenAI 補助）、`tomli`（設定, 3.11+ は tomllib で不要）
- mermaid はレポート内テキストとして埋め込み（描画はブラウザ側 or 事前SVG化を後で決定）

## 7. 段階実装計画
1. models / scanner / enrich（走査とハッシュ・信頼度）+ scan.json 出力
2. duplicates / versioning（重複・旧版の洗い出し）
3. classifier（ルールベース）+ proposer（改善後ツリー・移動計画CSV）
4. visualize（before/after mermaid + グラフJSON）+ report（HTML）
5. llm（Claude API 補助）を classifier / proposer に統合（フォールバック付き）
6. README・config.example・サンプルデータでの動作確認
